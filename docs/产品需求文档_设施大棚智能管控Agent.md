## 一、产品概述

### 1.1 产品定位

设施大棚智能管控 Agent 是一款基于 AI 的温室环境智能管理系统，通过融合传感器数据、天气预报、作物模型，为种植户提供精准的环境控制建议和自动化操作建议。

### 1.2 核心价值主张

| 价值 | 说明 |
|------|------|
| 省心 | 7×24小时自动监控，异常告警无需专人值守 |
| 省钱 | 水肥用量减少20-30%，能耗降低15-25% |
| 提质 | 环境控制精准，农产品品质一致性提升 |
| 增产 | 优化生长环境，产量提升10-20% |

### 1.3 目标用户

| 用户类型 | 规模 | 痛点 | 付费意愿 |
|----------|------|------|----------|
| 大型农业企业 | 100棚以上 | 管理效率低、人工成本高 | 强 |
| 中型合作社 | 30-100棚 | 技术缺乏、品质不稳 | 中 |
| 小型种植户 | 1-30棚 | 资金有限、技术门槛高 | 弱 |

### 2.2 功能详情

#### F01 环境数据采集（P0）

**描述**：系统接收来自各类传感器的数据

**数据源**：
- 空气温湿度传感器
- 光照传感器
- CO2浓度传感器
- 土壤EC/pH传感器
- 土壤温湿度传感器

**数据指标**：

| 指标 | 采集频率 | 数据格式 |
|------|----------|----------|
| 空气温度 | 1分钟 | ℃ |
| 空气湿度 | 1分钟 | % |
| 光照强度 | 1分钟 | lux |
| CO2浓度 | 5分钟 | ppm |
| 土壤EC | 10分钟 | mS/cm |
| 土壤pH | 10分钟 | - |
| 土壤温度 | 10分钟 | ℃ |
| 土壤含水率 | 10分钟 | % |

**验收标准**：
- [ ] 支持至少4种主流传感器品牌的数据接入
- [ ] 数据延迟不超过30秒
- [ ] 支持断网重连后数据补传

---

#### F02 环境异常告警（P0）

**描述**：当环境指标超出预设范围时，系统自动推送告警

**告警等级**：

| 等级 | 触发条件 | 通知方式 |
|------|----------|----------|
| 紧急 | 温度>40℃或<5℃ | 短信+APP推送+电话 |
| 警告 | 湿度>90%或<40% | APP推送 |
| 提醒 | 光照过强/过弱 | APP推送 |

**告警规则**：
- 告警阈值支持按作物类型自定义
- 告警可配置重复发送间隔（避免骚扰）
- 告警确认后消除或持续监控

**验收标准**：
- [ ] 告警触发延迟<10秒
- [ ] 支持至少3种通知方式
- [ ] 告警历史可查询

---

#### F03 控制建议生成（P0）

**描述**：Agent 根据当前环境数据、天气预报、作物模型，生成设备控制建议

**输入数据**：
1. 当前环境数据（温度/湿度/光照/CO2/土壤EC/pH）
2. 未来24-72小时天气预报
3. 作物当前生育期阶段
4. 历史操作记录

**输出建议**：
- 开启/关闭风机
- 升起/降下遮阳帘
- 开启/关闭补光灯
- 调整滴灌时长和频率
- 调整通风窗开度

**建议生成逻辑**：
```
IF 温度 > 30℃ AND 湿度 < 60%
THEN 建议：开启风机 + 启动滴灌（降低室温+补水）

IF 光照 < 10000lux AND 作物阶段 = 开花期
THEN 建议：开启补光灯（每日14小时光照）

IF 土壤EC > 4.0 AND 作物阶段 = 结果期
THEN 建议：暂停滴灌4小时 + 适当增加浇水稀释EC
```

**验收标准**：
- [ ] 建议生成延迟<5秒
- [ ] 建议准确率>85%（用户采纳率）
- [ ] 支持按作物类型定制建议逻辑

---

#### F04 手动控制设备（P1）

**描述**：用户可在APP上直接操作控制设备

**支持的设备**：
- 风机（开/关）
- 遮阳帘（升/降）
- 补光灯（开/关）
- 滴灌阀门（开/关）
- 通风窗（开度0-100%）
- CO2补充设备（开/关）

**操作记录**：所有手动操作自动记录到日志

**验收标准**：
- [ ] 操作响应延迟<1秒
- [ ] 支持批量控制
- [ ] 操作可撤销（5分钟内）

---

#### F05 自动控制执行（P1）

**描述**：用户配置自动化规则，系统自动执行控制

**自动化规则示例**：
```
规则1：温度>32℃ → 自动开启风机
规则2：光照>50000lux → 自动降下遮阳帘
规则3：土壤含水率<30% → 自动开启滴灌10分钟
规则4：CO2<350ppm → 自动开启CO2补充
```

**安全机制**：
- 自动控制与手动控制优先级：手动 > 自动
- 连续自动控制超过阈值（如风机连续开4小时）需人工确认
- 极端天气自动跳过（可配置）

**验收标准**：
- [ ] 规则配置灵活，支持AND/OR条件
- [ ] 规则可启用/禁用/编辑
- [ ] 支持"演习模式"（只发建议不执行）

---

#### F06 水肥一体化管理（P1）

**描述**：根据作物需肥规律和土壤状态，自动调整水肥配方

**水肥策略**：
- 根据作物生育期（苗期/花期/果期）自动调整EC目标值
- 根据土壤EC动态调整肥料浓度
- 根据天气（阴雨/晴天）调整灌溉策略

**支持的施肥模式**：
- 定量施肥（每亩X升）
- 周期施肥（每X小时施肥Y分钟）
- 应激施肥（根据传感器阈值触发）

**验收标准**：
- [ ] 支持至少3种作物类型的预置策略
- [ ] 水肥执行记录可追溯
- [ ] 支持EC/pH校准提醒

---

#### F07 多棚协同管理（P1）

**描述**：一个账号管理多个大棚

**功能**：
- 大棚列表总览（环境状态一览）
- 大棚分组（按区域/作物类型）
- 跨棚批量操作
- 跨棚数据对比分析

**验收标准**：
- [ ] 支持至少50个大棚的接入
- [ ] 支持跨棚数据汇总报表
- [ ] 支持权限分级（管理员/操作员/查看者）

## 三、用户故事

### Epic 1：环境监控

**故事1**：作为种植户，我希望随时查看大棚环境数据，不用每天亲自进棚检查，减少工作量。

**验收标准**：
- 打开APP 5秒内看到所有棚的温湿度
- 数据与传感器同步延迟<30秒

**故事2**：作为种植户，当大棚温度异常时我能立即收到通知，避免作物受损。

**验收标准**：
- 温度超阈值后10秒内收到告警
- 告警包含当前值、阈值、持续时长

---

### Epic 2：智能控制

**故事3**：作为种植户，我希望系统告诉我什么时候该浇水、施肥，而不是靠经验。

**验收标准**：
- 系统每日推送水肥建议
- 建议包含具体操作和时间

**故事4**：作为有经验的种植户，我希望某些简单操作能自动执行（如温度高自动开风机），省得我反复操作。

**验收标准**：
- 可配置自动化规则
- 自动执行前有3秒确认（可关闭）

**故事5**：作为合作社负责人，我希望在一个界面看到所有大棚的状态，不用一个个切换。

**验收标准**：
- 大棚列表页展示所有棚的温湿度
- 支持按区域/作物筛选

---

### Epic 3：数据分析

**故事6**：作为种植户，我希望看到历史环境数据，了解过去一周/一个月的变化趋势。

**验收标准**：
- 支持查看7天/30天/自定义时间范围的历史数据
- 支持数据导出（Excel格式）

**故事7**：作为合作社负责人，我希望看到每个棚的产量预测，提前规划采收和销售。

**验收标准**：
- 显示预计采收日期
- 显示预计产量范围

---

### Epic 4：增值服务

**故事8**：作为新手种植户，我希望能从系统获得种植技术指导，遇到问题有专家可咨询。

**验收标准**：
- 内置常见问题知识库
- 支持AI问答

**故事9**：作为合作社负责人，我希望能自动生成周报，向管理层汇报生产情况。

**验收标准**：
- 周报自动生成
- 支持自定义报告模板

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | <2秒（4G网络） |
| 数据刷新频率 | 实时（<30秒延迟） |
| 控制命令响应 | <1秒 |
| 系统可用性 | 99.5%（每月停机<4小时） |

### 4.2 安全要求

| 需求 | 说明 |
|------|------|
| 数据传输加密 | HTTPS/TLS 1.2+ |
| 用户认证 | 支持手机号+验证码登录 |
| 权限控制 | 支持角色分级（管理员/操作员/查看者） |
| 数据备份 | 每日自动备份，保留30天 |

### 4.3 可用性要求

| 需求 | 说明 |
|------|------|
| 支持平台 | iOS 13+/Android 8+ |
| 离线支持 | 传感器数据本地缓存，网络恢复后同步 |
| 操作简化 | 关键操作不超过3步 |

### 4.4 兼容性要求

| 需求 | 说明 |
|------|------|
| 传感器协议 | 支持 Modbus RTU/TCP、HTTP API |
| 设备对接 | 预留串口/网口扩展能力 |
| 系统集成 | 提供API供第三方系统调用 |

## 五、产品路线图

### Phase 1：MVP（0-6个月）

**目标**：验证核心价值，签约首批付费客户

**功能范围**：
- F01 环境数据采集
- F02 环境异常告警
- F03 控制建议生成
- F04 手动控制设备

**发布条件**：
- 至少3个种子客户
- 建议采纳率>70%
- 无P0级Bug

---

### Phase 2：产品完善（6-12个月）

**目标**：完善功能，扩展客户群体

**新增功能**：
- F05 自动控制执行
- F06 水肥一体化管理
- F07 多棚协同管理

**发布条件**：
- 客户数>30
- 月流水>30万
- 核心功能稳定运行3个月

---

### Phase 3：规模化（12-24个月）

**目标**：进入更多省份，建立渠道体系

**新增功能**：
- F08 历史数据查询
- F09 产量预测
- F10 种植日志
- F11 远程专家咨询
- F12 报表生成

**发布条件**：
- 覆盖省份>5
- 年收入>1000万
